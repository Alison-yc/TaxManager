> 用途：在 Cursor / 其他 IDE 中将本文件作为项目说明与任务约束的补充，便于在多轮对话中保持方向一致。**随功能迭代请及时更新本文。**

---

## 1. 项目是什么

- **定位**：对标 **「全国统一规范电子税务局」一类站点**流程的 **内部测试 / 原型系统**（并非官方产品，不涉及与真实税局生产系统直连申报）。
- **目标形态**：**登录** → **首页** → **顶栏/我要查询** 进入 **申报信息查询页** → **检索列表** → **种类** 进入 **预览** → **导出 PDF**（及下载 Excel）。设计者提供的 **PNG 整页截图为类似 Figma 的目标版式**，**实现时用 HTML/CSS 还原外观，不在页面中用 `<img>` 嵌入整张截图**；`public/assets/etax/` 仅存档或给其他场景用，核心业务页可不依赖大图。
- **当前前端**：**React + Vite + TypeScript**，**SPA**，无自建后端进程。
- **数据与账号**：**Supabase**（Auth + Postgres），**无独立业务服务器**。
- **静态资源**：**`public/assets/etax/`**（电子税务局视觉原型图，随构建发布；代码中通过 `import.meta.env.BASE_URL` 引用）。

## 2. 页面跳转与交互（真实能力 vs 仅占位）

顺序与 **必须实现** 的路径如下（其他入口 **仅 hover 高亮**，**无路由/无业务**）：

| 步骤 | 页面 / 区域 | 触发 | 目标 / 行为 |
|------|----------------|------|-------------|
| A | **登录页** | 提交表单 | 校验 **账号**（可自动拼邮箱后缀，`authAccount.ts`）+ **密码** → `signInWithPassword`。 |
| — | ↑ 第一项「一串数字」等 | — | **不提交、不调接口**，仅存本地状态用于还原版式（装饰字段）。 |
| B | **首页 `/`** | 顶栏或其他 **非核心**菜单 | **无跳转**（或仅 hover 样式）。 |
| — | **首页 `/`** · 我要查询区 | **点击「申报信息查询」整块（核心图区域）** | **跳转 `/query`**。 |
| C | **申报信息查询 `/query`** | 筛选 + **查询** | 可先 **全量拉取** `form_data`（后续再对齐枚举）；首项筛选若无枚举可先 **占位禁用**。 |
| — | ↑ 表格行 **「种类」列**（链接） | 点击 | **跳转 `/record/:id`**（预览）。 |
| — | ↑ **「导出」** | 点击 | **跳转 `/record/:id?pdf=1`**，预览页 **自动执行一次** 现有 **导出 PDF** 逻辑后去掉 query（避免重复）。 |
| D | **预览 `/record/:id`** | 页面内 **导出 PDF / 下载 Excel** | 沿用 `excelExport.ts`；返回列表一般用 **« 返回申报信息查询** → `/query`。 |

**不在本期赋能力的**：办税服务厅等多余顶栏项、报税 Banner 内的假按钮链路、信息公开等——**只允许 hover**，**不出现 `Link`/`navigate`**。

## 4. 架构与约定

| 层级 | 说明 |
|------|------|
| 路由 | **`/`** 首页；**`/query`** 申报信息查询；**`/record/:id`** 预览。`basename = import.meta.env.BASE_URL`。 |
| 变量 | `VITE_SUPABASE_*`、`VITE_AUTH_EMAIL_SUFFIX`（账号自动拼后缀）。 |
| Excel | `xlsx` 解析用户上传文件；解析后的网格与索引字段写入 Supabase。 |
| PDF | `html2canvas` + `jsPDF`；`?pdf=1` 触发自动导出。 |

---

## 5. 数据模型（Supabase）

- **表**：`form_data`，`content` 为 `jsonb`；**`importVersion: 2`** 含 `grid`、`merges` 等，供预览与导出。
- **共享**：默认 **已登录** 可读写的 RLS（与产品一致时需收紧则单独立项）。

## 7. 实现约束（给助手）
