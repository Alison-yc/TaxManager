# 报税/表单数据管理系统（TaxManager）— 二期扩展说明与报价

**文档版本**：v0.4（二期）  
**相对上版**：v0.3 已完成 Excel 导入与单链路预览导出；本期扩展 PDF 多源导入与多类型导出。  
**开发投入**：约 **3 个工作日**（集中开发）。

---

## 一、本期相对上期的增量价值

在**不增加自建服务器与运维**前提下，在 v0.3 基础上新增：

1. **四类 PDF 导入**（账户中心统一入口，导入后进入对应列表）  
   - 数电 **发票** PDF  
   - **申报表** PDF  
   - **财务报表** PDF  
   - **税收完税证明** PDF  

2. **五类文件导出/下载能力**  
   - 申报类：**Excel 网格 → PDF**（沿用 v0.3 能力）  
   - 申报/财报/发票/完税证明：**原 PDF 预览 + 下载**（导入什么，预览/导出什么）  
   - 发票：**全量查询 Excel 导出**（基于官方模板样式填表）  

3. **业务页面与导航**  
   - 门户「我要办税 / 我要查询」可跳转：申报查询、财报查询、发票查询统计/全量、完税证明查询等  
   - 首页热门服务「开具税收完税证明」可跳转  
   - 各列表页：检索条件、预览、导出、部分列表支持双击删除后重新导入  

4. **后台（Supabase）扩展**  
   - 新表：`invoice_records`、`tax_payment_certificate_records` 等  
   - 私有桶 `imported-docs` 及分目录存储策略  
   - 配套 migration 与 RLS（按用户隔离）  

---

## 二、技术实现要点（便于验收对齐）

| 模块 | 说明 |
|------|------|
| PDF 解析 | 浏览器端 pdf.js 抽文本 + 针对数电票/完税证明等版式的解析规则 |
| PDF 存储 | 原文件上传 Supabase Storage，列表/预览/导出走同一路径 |
| 发票 Excel 导出 | 官方 xlsx 模板 + ExcelJS，导出条数为用户勾选/查询结果 |
| 完税证明 | 1 份 PDF = 1 条列表记录；金额取 PDF 合计；预览/导出为原 PDF |
| 完税证明列表 | 查询方式与列联动（属期 / 缴退款时间）；金额列双击可删记录 |

## 三、交付范围与边界

**包含（本期）**

- 上述 4 类 PDF 导入、5 类导出/下载、相关列表/预览/路由/菜单

- 与 v0.3 一致的登录与静态站点发布方式

- 多角色权限、审计、大量并发性能优化  
- **超出 1 轮优化范围的修改、以及上线后的新功能需求**（需另行报价）

## 五、验收建议

1. 每类 PDF 各提供 **1～2 份样例**，走通：导入 → 列表可见 → 预览 → 导出/下载。  
2. 发票 Excel 导出：勾选 N 条，文件中仅 N 行数据（非模板样例行）。  
3. 完税证明：1 PDF 对应 1 条；金额为合计；删后可重导。  
4. 赠送的 1 轮优化：以双方确认的**修改清单**为准，清单内问题关闭即视为该轮完成。
